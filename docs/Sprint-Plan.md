# 8-Week Agile Sprint Plan

## Sprint 1 - Foundation and Product Definition, Weeks 1-2

### Goal
Set up the project structure, documentation, frontend skeleton, and backend skeleton.

### Stories
- As a product owner, I want a PRD so the MVP scope is clear.
- As a developer, I want a clean repo structure so the project is easy to maintain.
- As a user, I want a basic form so I can enter compute requirements.
- As a developer, I want a health check API so backend readiness can be validated.

### Deliverables
README, SOU, PRD, architecture, API spec, backend skeleton, frontend skeleton.

## Sprint 2 - Pricing Data and Backend Logic, Weeks 3-4

### Goal
Implement seed pricing data and the comparison API.

### Stories
- As a user, I want seed pricing data so I can test comparisons.
- As a user, I want to submit CPU and memory requirements and receive results.
- As a developer, I want normalized pricing records so providers can be compared fairly.

### Deliverables
Seed dataset, comparison service, POST compare endpoint.

## Sprint 3 - UI Results and Matching Quality, Weeks 5-6

### Goal
Build a usable UI and improve result clarity.

### Stories
- As a user, I want results in a table so I can compare providers quickly.
- As a user, I want match quality so I know whether the result is exact or approximate.
- As a user, I want monthly pricing so I can estimate run-rate cost.

### Deliverables
Frontend connected to backend, results table, match quality, loading/error states.

## Sprint 4 - Polish and Portfolio Readiness, Weeks 7-8

### Goal
Prepare the MVP for GitHub showcase and demo use.

### Stories
- As a reviewer, I want clear setup instructions so I can run the app.
- As a hiring manager, I want professional docs so I can assess product thinking.
- As a developer, I want Docker support so setup is easy.

### Deliverables
Docker Compose, final docs, roadmap, demo script, GitHub-ready README.
